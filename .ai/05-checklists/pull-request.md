# Pull Request Checklist
- [ ] Title + description follow Conventional Commits.
- [ ] Scope is small and focused; no unrelated changes.
- [ ] Lint + types + tests pass locally and in CI.
- [ ] No secrets, no `console.*`, no stray `any` without justification.
- [ ] Public API/contract changes documented.
- [ ] Reviewer can run it: setup notes if non-trivial.
- [ ] Linked to issue/ticket; business rules referenced.
